## 1. 현황 요약

| 항목 | 현재 상태 | 평가 |
|------|-----------|------|
| 전체 테마 | 다크 모드 중심, 카드 기반 대시보드 | ✅ 방향 적합 |
| 색상 일관성 | 위젯마다 배경색이 다름 (갈색/남색/다크그린 혼재) | ⚠️ 혼란 유발 |

## 2. 정보 구조(IA) 개선

### 2-1. 섹션 앵커 네비게이션 추가

**문제:** 스크롤이 길어 원하는 위젯까지 이동하려면 무조건 스크롤해야 함.

**개선안:**
```
헤더 하단 또는 사이드바에 섹션 퀵점프 메뉴 추가
예) [블로그] [가계부] [일정] [운세] [복권] [증시] [행사] [부동산]
```

- 스크롤 위치에 따라 현재 섹션이 하이라이트되는 **Sticky 섹션 탭** 또는 **사이드 도트 인디케이터** 권장
- 모바일에서는 하단 고정 탭바(Bottom Navigation Bar)로 전환

---

### 2-2. 위젯 우선순위 그룹화

**문제:** 8개 위젯이 동등한 시각적 비중으로 나열되어 있어 "이 사이트가 뭘 잘하는 사이트인지" 즉시 파악 어려움.

**개선안:**

```
[Primary — 상단 2열]
운세 & 카드 / 로또 복권

[Secondary — 중단]
증시 포털 (가로 전체폭)

[Tertiary — 하단 2열]
블로그 / 가계부 / 일정 / 부동산

[외부 콘텐츠 — 최하단]
행사·축제
```

- 사용자가 가장 자주 쓸 "재미" 콘텐츠(운세, 복권)를 상단에 배치
- 정보성 콘텐츠(증시, 부동산)는 중·하단 배치

## 3. 레이아웃 & 그리드 개선

### 3-1. 위젯 높이 통일

**문제:** 카드마다 내부 콘텐츠 양에 따라 높이가 제각각이어서 시각적 리듬이 깨짐.

**개선안:**
- 상단 로그인 안내 카드 3개는 동일한 최소 높이(min-height) 설정
- 콘텐츠가 적은 카드는 `stretch` 또는 `placeholder` 패턴으로 채움

---

### 3-2. 카드 간격(Gap) 조정

**문제:** 카드 사이 여백이 좁아 위젯 경계가 불명확함.

**개선안:**
```css
/* 현재 추정 */
gap: 12px;

/* 개선 */
gap: 20px; /* 데스크탑 */
gap: 12px; /* 모바일 */
```

---

### 3-3. 전체 최대 너비 제한

**문제:** 4K 등 넓은 화면에서 콘텐츠가 무한정 늘어남.

**개선안:**
```css
.dashboard-container {
  max-width: 1600px;
  margin: 0 auto;
  padding: 0 24px;
}
```

## 4. 타이포그래피 개선

### 4-1. 폰트 위계 명확화

**문제:** 카드 상단의 카테고리명(예: "블로그", "가계부")과 타이틀("최신 아티클")의 시각적 차이가 작음.

**개선안:**

| 역할 | 현재 | 개선 |
|------|------|------|
| 카테고리 레이블 | 소문자, 색상 강조 | `font-size: 11px`, `letter-spacing: 0.08em`, `text-transform: uppercase` |
| 위젯 타이틀 | 비슷한 크기 | `font-size: 20px`, `font-weight: 700` |
| 본문/설명 | - | `font-size: 14px`, `color: var(--text-muted)` |

---

### 4-2. 한국어 최적화 폰트 적용

**현재:** 기본 시스템 폰트 추정

**권장 폰트 (Google Fonts):**

```html
<!-- 가독성과 개성 모두 잡는 조합 -->
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+KR:wght@400;500;700&family=Pretendard:wght@400;600;700&display=swap" rel="stylesheet">
```

- **Pretendard** — 현대적이고 깔끔한 한국어 UI 폰트 (가장 추천)
- **Noto Sans KR** — 안정적이고 범용적인 선택

## 5. 컬러 시스템 개선

### 5-1. 현재 컬러 문제

위젯별로 배경색이 개별 지정되어 있어 일관성 부재:

| 위젯 | 현재 배경 추정 |
|------|----------------|
| 블로그 | 다크 브라운 (`#2a1a0e`) |
| 가계부 | 다크 네이비 (`#0e1a2a`) |
| 일정 | 다크 틸 (`#0a1f20`) |
| 운세 | 다크 퍼플 (`#1a0e2a`) |
| 복권 | 다크 옐로우브라운 (`#1f1a00`) |
| 증시 | 다크 레드브라운 (`#1f0e0e`) |

### 5-2. 개선된 컬러 팔레트 제안

사이트의 "생활 정보 + 재미 요소" 성격에 맞게, **딥 다크 + 네온 포인트** 테마 권장.

```css
:root {
  /* === 기본 배경 === */
  --bg-base:        #0d0f14;   /* 메인 배경 — 거의 검정에 가까운 네이비 */
  --bg-card:        #161921;   /* 카드 배경 */
  --bg-card-hover:  #1e2230;   /* 카드 호버 */
  --bg-elevated:    #252a38;   /* 입력창, 선택된 탭 */

  /* === 텍스트 === */
  --text-primary:   #f0f2f8;   /* 주요 텍스트 */
  --text-secondary: #9aa3b8;   /* 보조 텍스트 */
  --text-muted:     #5a6278;   /* 비활성 텍스트 */

  /* === 브랜드 포인트 === */
  --accent-primary:  #7c6af7;  /* 메인 포인트 — 소프트 바이올렛 */
  --accent-secondary:#f7a35c;  /* 서브 포인트 — 따뜻한 오렌지 (운세/복권) */
  --accent-teal:     #3dd9c0;  /* 정보성 포인트 — 틸 (증시 상승) */

  /* === 상태 색상 === */
  --color-up:    #26d87c;   /* 주가 상승 */
  --color-down:  #f05c6e;   /* 주가 하락 */
  --color-info:  #5cabf2;   /* 정보 */
  --color-warn:  #f7c137;   /* 경고 */

  /* === 테두리 === */
  --border-subtle:  rgba(255,255,255,0.06);
  --border-default: rgba(255,255,255,0.12);
}
```

### 5-3. 위젯별 포인트 컬러 가이드

각 위젯은 동일한 `--bg-card`를 사용하되, 카테고리 레이블과 아이콘에만 포인트 색상 적용:

| 위젯 | 레이블/아이콘 색상 |
|------|--------------------|
| 블로그 | `--accent-secondary` (오렌지) |
| 가계부 | `--accent-teal` (틸) |
| 일정 | `--color-info` (블루) |
| 운세 | `--accent-primary` (바이올렛) |
| 복권 | `--color-warn` (골드) |
| 증시 | `--color-down` (레드) |
| 행사 | `#4cce8a` (그린) |
| 부동산 | `#5cabf2` (스카이블루) |

## 6. 인터랙션 & 애니메이션 개선

### 6-1. 카드 호버 효과

**문제:** 카드에 호버 시 시각적 피드백이 약함.

**개선안:**
```css
.widget-card {
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}
.widget-card:hover {
  transform: translateY(-2px);
  box-shadow: 0 8px 32px rgba(0, 0, 0, 0.4);
}
```

---

### 6-2. 스켈레톤 로딩 UI

**문제:** 증시, 복권 등 API 데이터 로딩 시 빈 화면 또는 로딩 스피너만 표시.

**개선안:**
- 데이터 로딩 중에는 **Skeleton UI** 표시 (콘텐츠 형태를 흉내낸 플레이스홀더)
- 라이브러리: `react-loading-skeleton` 또는 CSS만으로 구현 가능

```css
.skeleton {
  background: linear-gradient(90deg, var(--bg-elevated) 25%, var(--bg-card-hover) 50%, var(--bg-elevated) 75%);
  background-size: 200% 100%;
  animation: shimmer 1.5s infinite;
}
@keyframes shimmer {
  0% { background-position: 200% 0; }
  100% { background-position: -200% 0; }
}
```

---

### 6-3. 페이지 진입 애니메이션

**문제:** 카드들이 한꺼번에 나타남.

**개선안:**
- 카드들이 순차적으로(stagger) 페이드인되도록 처리
- 라이브러리: **Framer Motion** (`staggerChildren` 활용)

```jsx
const containerVariants = {
  hidden: {},
  visible: { transition: { staggerChildren: 0.08 } }
};
const cardVariants = {
  hidden: { opacity: 0, y: 16 },
  visible: { opacity: 1, y: 0, transition: { duration: 0.4 } }
};
```

## 7. 인증 / 게스트 경험 개선

### 7-1. 로그인 버튼 일관성

**문제:** 위젯마다 로그인 버튼 디자인이 다름 (일부는 채운 버튼, 일부는 아이콘 포함 버튼).

**개선안:**
- 로그인 CTA 버튼 디자인 통일
- `variant="outline"` 스타일로 통일하거나, 헤더의 로그인 아이콘과 시각적으로 연결

## 8. 접근성(Accessibility) 개선

### 8-1. 아이콘 버튼 레이블 추가

```jsx
// 개선 전
<button><RefreshIcon /></button>

// 개선 후
<button aria-label="데이터 새로고침"><RefreshIcon /></button>
```

---

### 8-2. 색상 대비 확인

- 다크 배경에서 `--text-muted` 색상이 WCAG AA 기준(4.5:1) 미달 가능성 있음
- 특히 카드 내 설명 텍스트(`#5a6278` 추정) 재검토 필요
- 도구: [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)

---

### 8-3. 포커스 스타일 명확화

```css
:focus-visible {
  outline: 2px solid var(--accent-primary);
  outline-offset: 2px;
  border-radius: 4px;
}
```

---

### 8-4. 로또 번호 색상 의존 문제

**문제:** 로또 번호 구분을 색상으로만 표현 (색맹 사용자 불편).

**개선안:**
- 번호 위에 범위 텍스트(예: `1-10`, `11-20`) 툴팁 또는 접근 가능한 레이블 추가

## 9. 모바일 반응형 개선

### 9-1. 하단 고정 탭바 (Bottom Navigation)

**문제:** 모바일에서 위젯 간 이동이 전적으로 스크롤에 의존.

**개선안:**
```
[홈] [운세] [복권] [증시] [더보기]
 🏠    🔮     🎱    📈     ···
```

- 모바일에서 `position: fixed; bottom: 0` 탭바 추가
- 각 탭 클릭 시 해당 섹션으로 스무스 스크롤

---

### 9-2. 행사·축제 카드 모바일 스와이프

**문제:** 가로 스크롤 카드가 모바일에서 터치 스와이프 경험 최적화 미흡.

**개선안:**
- **Swiper.js** 또는 **Embla Carousel** 도입으로 터치 스와이프 + 페이지네이션 인디케이터 추가

---

### 9-3. 증시 테이블 모바일 최적화

**문제:** 증시 카드의 정보 밀도가 높아 390px 너비에서 텍스트가 겹침.

**개선안:**
- 모바일에서는 종목명 + 현재가 + 등락률만 표시, 나머지는 토글로 숨기기
- 가격 범위 바는 모바일에서 숨김 처리
